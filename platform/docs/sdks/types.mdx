## Overview

The Types SDK provides comprehensive TypeScript type definitions for all Flowglad data structures. This package ensures type safety when working with Flowglad's API and SDKs.

## Installation

<CodeGroup>

```bash npm
npm install @flowglad/types
```

```bash yarn
yarn add @flowglad/types
```

```bash bun
bun add @flowglad/types
```

</CodeGroup>

## Key Features

- **Comprehensive Type Coverage**: Types for all Flowglad entities
- **Enum Definitions**: Type-safe enums for statuses and categories
- **Interface Exports**: Clean interfaces for data structures
- **Zero Dependencies**: Lightweight type-only package
- **IDE Autocomplete**: Enhanced developer experience with IntelliSense

## Common Patterns

### Type-Safe Function Parameters

```typescript
import type { Subscription, SubscriptionStatus } from '@flowglad/types'

function filterActiveSubscriptions(
  subscriptions: Subscription[]
): Subscription[] {
  return subscriptions.filter(
    (sub) => sub.status === SubscriptionStatus.Active
  )
}
```

### Type Guards

```typescript
import type { Payment } from '@flowglad/types'
import { PaymentStatus } from '@flowglad/types'

function isPaymentSuccessful(payment: Payment): boolean {
  return (
    payment.status === PaymentStatus.Succeeded ||
    payment.status === PaymentStatus.Processing
  )
}

function getSuccessfulPayments(payments: Payment[]): Payment[] {
  return payments.filter(isPaymentSuccessful)
}
```

### Custom Type Composition

```typescript
import type { Customer, Subscription, PaymentMethod } from '@flowglad/types'

interface CustomerBillingInfo {
  customer: Customer
  activeSubscriptions: Subscription[]
  paymentMethods: PaymentMethod[]
  totalSpent: number
}

function createBillingInfo(
  customer: Customer,
  subscriptions: Subscription[],
  paymentMethods: PaymentMethod[],
  totalSpent: number
): CustomerBillingInfo {
  return {
    customer,
    activeSubscriptions: subscriptions.filter(
      (sub) => sub.status === 'active'
    ),
    paymentMethods,
    totalSpent,
  }
}
```

### API Response Typing

```typescript
import type { Subscription, Payment, Invoice } from '@flowglad/types'

interface BillingDashboardData {
  subscriptions: Subscription[]
  recentPayments: Payment[]
  unpaidInvoices: Invoice[]
  monthlyRevenue: number
}

async function fetchBillingDashboard(): Promise<BillingDashboardData> {
  const response = await fetch('/api/billing/dashboard')
  const data: BillingDashboardData = await response.json()
  return data
}
```

### Type-Safe State Management

```typescript
import type { Customer, Subscription } from '@flowglad/types'
import { create } from 'zustand'

interface BillingStore {
  customer: Customer | null
  subscriptions: Subscription[]
  setCustomer: (customer: Customer) => void
  setSubscriptions: (subscriptions: Subscription[]) => void
}

export const useBillingStore = create<BillingStore>((set) => ({
  customer: null,
  subscriptions: [],
  setCustomer: (customer) => set({ customer }),
  setSubscriptions: (subscriptions) => set({ subscriptions }),
}))
```
